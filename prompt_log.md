阶段二：发现并处理 Bug
用户：运行测试后输出解数量为 0，日志显示回溯过程中所有皇后放置尝试均被拒绝，需分析原因并修复。执行命令：pytest tests/ -v。AI：错误根源是冲突检测逻辑严重缺失，is_safe()方法仅判断了当前行是否已有皇后，未校验列和对角线冲突。修复方案：
列冲突检查：遍历已放置皇后的列，判断目标列是否已被占用；
对角线冲突检查：通过abs(当前行 - 目标行) == abs(当前列 - 目标列)判断是否在同一对角线；
重构is_safe(self, queens: list[int], row: int, col: int) -> bool方法：
遍历queens中已放置的皇后位置(r, c)；
若c == col（列冲突）或abs(r - row) == abs(c - col)（对角线冲突），返回False；
无冲突则返回True。
修复后重新运行测试，4 皇后返回 2 个解、8 皇后返回 92 个解，测试通过。
